Proposta Técnica para Aplicação de Treinamento (Musculação e Ciclismo)
1. Visão Geral do Produto

O objetivo é construir uma aplicação para auxiliar na preparação física de um atleta que combina musculação e ciclismo. O sistema deve consolidar dados de atividades, analisar sinais de fadiga e progresso, e gerar recomendações diárias de treino usando uma IA (modelo via API). A aplicação será modular para evoluir de um uso pessoal (MVP) para um produto escalável.

1.1 Escopo principal
Integrar dados de plataformas populares (Strava, TrainingPeaks, Garmin) para obter métricas de treinos, sono e recuperação.
Permitir o registro manual de treinos de musculação, dores musculares, descanso e qualidade do sono.
Analisar a carga de treino (CTL/ATL/TSB) e sugerir treinos futuros via IA.
Exportar e sincronizar treinos planejados para as plataformas compatíveis.
Possibilitar a execução de rotinas automatizadas via Claude Code Routines (ex.: um fluxo diário que importa dados, calcula indicadores e programa o próximo treino).
2. Limitações e Considerações de Integração
2.1 Garmin Connect
A Garmin disponibiliza suas APIs através do Garmin Connect Developer Program. O acesso é restrito a entidades legais (empresas, universidades etc.); aplicações de uso pessoal são rejeitadas.
É necessário solicitar acesso como empresa, fornecer site público e política de privacidade e passar por verificação da Garmin.
Para uso pessoal, é inviável usar a API oficial. Alternativas:
Utilizar um serviço agregador como Open Wearables (openwearables.io) que integra Garmin via autorização OAuth para empresas (ainda exige credenciais empresariais, mas abstrai a API).
Dependendo dos dados desejados, sincronizar dados por meio de Strava (Garmin envia automaticamente treinos para Strava) e importar a partir de lá.
2.2 TrainingPeaks
O API oficial da TrainingPeaks é apenas para desenvolvedores aprovados e não está disponível para uso pessoal. É necessário enviar um e‑mail explicando o caso de uso (api@trainingpeaks.com
) e aguardar aprovação.
Existe um projeto open‑source chamado trainingpeaks‑mcp que implementa um servidor MCP usando autenticação via cookie do navegador e permite consultar e manipular treinos sem aprovação oficial. Ele oferece várias ferramentas (listar, criar, editar treinos, acessar CTL/ATL/TSB, logar métricas de saúde, etc.). Ele não requer aprovação da API, mas seu uso pode violar os termos de uso da TrainingPeaks; portanto, recomenda‑se apenas para uso pessoal e com cautela.
2.3 Strava
O Strava API é público para desenvolvedores e oferece acesso via OAuth 2.0. Novos aplicativos ficam em "Single Player Mode", permitindo acesso apenas aos dados do próprio atleta. Para suportar múltiplos usuários, é necessário solicitar ampliação de capacidade ao Strava.
Há limite de 200 requisições por 15 min e 2 000 por dia (100/1 000 para endpoints "non‑upload").
O Strava é adequado para importar atividades de ciclismo e corrida, além de métricas como frequência cardíaca, potência e cadência.
2.4 Alternativas de Integração
Terra API ou Open Wearables API: APIs unificadas que conectam múltiplos dispositivos (Garmin, Suunto, Polar, TrainingPeaks etc.) mediante cadastro e fornecem dados normalizados. Podem ser usadas para centralizar ingestão de dados, mas exigem contrato comercial.
Google Fit/Apple Health**: para capturar dados de sono, batimentos e peso.
Planilha própria (Google Sheets): como backup para armazenamento de dados e visualização; excelente para agentes MCP.

                │                               │

           │ Banco de Dados │                │ Serviços AI    │
           │  (PostgreSQL)  │                │ (OpenAI API)   │

Frontend: pode ser um aplicativo web (React/Vue) ou móvel (React Native/Flutter) para visualização dos treinos e registro manual.
API Gateway/Backend: expõe endpoints REST/GraphQL para o frontend, gerencia autenticação e integrações. Realiza a ponte OAuth com Strava, possivelmente com TrainingPeaks via MCP e com a IA. Sugere usar Python (FastAPI) ou Node.js (NestJS).
Servidor MCP para TrainingPeaks: opcional; roda separadamente e é integrado como um microserviço. O repositório trainingpeaks-mcp fornece uma série de ferramentas para manipular dados do TP.
Banco de dados: PostgreSQL para armazenar usuários, treinos, sessões de musculação, métricas de saúde, e as recomendações produzidas pela IA.
Serviços de IA: integrar via API ao modelo (OpenAI, Anthropic, ou outro) para análise dos dados e geração de recomendações de treino.
Routines (Claude Code): para automatizar importações e gerações diárias de treinos. As rotinas são programadas no Claude Code e executam em cloud, podendo ser disparadas por horário ou via webhook.
3.2 Segurança e Autenticação
OAuth 2.0: implementar fluxo de autorização para Strava e, se aprovado, para TrainingPeaks e Garmin. Armazenar tokens de forma criptografada (por exemplo, usando o django-encrypted-fields ou vault).
HTTPS: todas as comunicações devem ser feitas sobre TLS. Utilizar certificados (Let's Encrypt).
Controle de Acesso: autenticação JWT ou sessão; aplicar RBAC caso futuramente haja múltiplos usuários ou coaches.

5. Integrações Detalhadas
5.1 Integração com Strava
Registrar o app no Strava Developers
.
Configurar o redirect URI para o backend (ex.: https://seuapp.com/api/auth/strava/callback).
Implementar fluxo OAuth (Authorization Code) para obter access_token e refresh_token. Tokens devem ser atualizados conforme expiração.
Consumir endpoints da API:
GET /athlete/activities: lista de atividades do atleta (corrida, ciclismo, natação).
GET /activities/{id}: detalhes, laps, zonas de frequência cardíaca, potência e cadência.
POST /activities: se quiser exportar treinos planejados (Strava permite upload de atividades).
Respeitar limites: 200 requisições/15 min, 2 000/dia (100/1 000 para endpoints de leitura).
Athlete Capacity: novo app inicia em “Single Player Mode”, permitindo apenas 1 atleta (você).
5.2 Integração com TrainingPeaks via MCP
Clonar o repositório trainingpeaks-mcp e executar o servidor conforme instruções. Esse servidor utiliza cookie de sessão para acessar sua conta e não requer aprovação da API.
Executar o comando tp-mcp auth --from-browser para extrair o cookie da sessão logada no browser.
O servidor expõe ferramentas (via JSON RPC ou HTTP) que podem ser chamadas pelo backend ou diretamente por agentes de IA. Exemplos:
tp_get_workouts para listar treinos num intervalo
tp_create_workout para criar um treino estruturado
tp_analyze_workout para obter métricas de CTL/ATL/TSB
tp_log_metrics para registrar métricas de saúde (peso, sono)
Incluir uma camada adaptadora no backend para consumir essas ferramentas.
Cuidados: como não é um acesso oficial, use apenas para fins pessoais e em conformidade com termos de uso.
5.3 Integração com Garmin (quando viável)
Só possível mediante aceitação no Garmin Connect Developer Program. Requer empresa registrada, domínio e política de privacidade.
Caso obtida a aprovação, seguir passos: solicitar credenciais (client ID/secret), configurar endpoints de webhook para receber dados e implementar OAuth.
Para uso pessoal, recomenda‑se capturar treinos da Garmin via Strava (sincronização automática) ou via plataformas unificadas (Terra/Open Wearables).
5.4 Registro Manual e Outras Fontes
Implementar formulários para registrar treinos de musculação: data, exercício, séries, repetições, carga, RPE.
Registrar métricas subjetivas: dores, fadiga, qualidade do sono, sensação de recuperação.
Sincronizar dados de sono/peso via Google Fit/Apple Health ou inserir manualmente.

7. Fluxo Diário da IA
Importação de dados: rotina (cron ou Claude Routines) chama o backend para importar dados recentes da Strava e TrainingPeaks (via MCP).
Atualização de métricas: calcular CTL/ATL/TSB (fitness, fadiga e forma), distância semanal e cargas de musculação, e atualizar banco de dados.
Análise: enviar resumo dos dados (treinos, métricas, estado de recuperação) para a API de IA.
Geração de recomendação: a IA retorna o treino sugerido (ex.: tipo de sessão, duração, intensidade, exercícios de musculação).
Planejamento: backend grava a recomendação na tabela recommendations e, opcionalmente, agenda o treino no TrainingPeaks via MCP ou exporta para Strava (upload de atividade programada).
Notificação ao usuário: enviar notificação push/e‑mail ou exibir no app.
Feedback: após a realização do treino, o usuário registra como se sentiu; esses dados alimentam a próxima análise.
8. Prompt Inicial da IA (exemplo)

Para gerar recomendações de treino, enviaremos à API de IA um prompt estruturado contendo dados do usuário e métricas recentes. Exemplo de prompt (em inglês para modelos atuais):

System: You are a professional endurance coach and strength trainer. You create daily training sessions for an athlete who cycles and lifts weights.

User data:

Age: 34, sex: male, weight: 78 kg, height: 1.78 m.
FTP: 260 W, max heart rate: 186 bpm.
Goals: Gran Fondo on August 20, 2026 (16 weeks away), increase squat strength.
Weekly availability: cycling Tue/Thu/Sat, strength Mon/Fri, rest Wed/Sun.

Recent workouts (past 7 days):

Tue: 2h endurance ride, 120 TSS; Wed: 45 min mobility; Thu: 4×6 min at threshold, 100 TSS; Fri: full-body strength (bench press, deadlift)…
Total weekly TSS: 450, ATL 78, CTL 72, TSB −6.
Subjective fatigue: 7/10; sleep quality: 8/10.

Instructions: Based on the athlete’s current fatigue (TSB −6) and upcoming event, propose a training session for tomorrow. Include warm‑up, main set, and cool‑down for cycling, and suggest accessory strength exercises. Explain your rationale and adjust intensity if the athlete reports high fatigue.

O backend formata esse prompt dinamicamente com dados reais do usuário e envia à API de IA. A resposta é então parseada e armazenada.
